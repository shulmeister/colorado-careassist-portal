# Gigi Tech Team Management Commands

## Daily Operations

### Engineering Standup
```
"Gigi, run tech team daily standup:
1. Message each engineering team member for status
2. Collect yesterday's commits and today's priorities
3. Check build status and test coverage updates
4. Identify any technical blockers or dependencies
5. Compile engineering status report with metrics"
```

### Build & Performance Check
```
"Gigi, pull engineering health dashboard:
1. Check build success rate (target: >95%)
2. Review test coverage across all repos (target: >80%)
3. Monitor Lighthouse performance scores (target: >90)
4. Check production error rates (target: <0.1%)
5. Alert on any metrics below threshold"
```

## Protocol Enforcement

### Code Review Validation
```
"Gigi, validate code review process:
1. Check all PRs have 2+ reviewers assigned
2. Monitor review turnaround time (<24 hours)
3. Verify test coverage before merge approval
4. Ensure design docs exist for new features
5. Flag stale PRs for Head of Engineering attention"
```

### Deployment Readiness
```
"Gigi, validate deployment readiness:
1. Confirm 5-step protocol completion
2. Verify code review and QA approval
3. Check performance benchmarks maintained
4. Validate test coverage meets threshold
5. Get Head of Engineering LGTM confirmation"
```

## Team Accountability

### Weekly Sprint Review
```
"Gigi, conduct tech team sprint review:
1. Compare sprint commitments vs. actual completion
2. Analyze build quality and error rates by engineer
3. Review test coverage contributions
4. Identify velocity trends and blockers
5. Generate engineering performance summary"
```

### Technical Debt Assessment
```
"Gigi, assess technical debt status:
1. Review accumulated debt items by priority
2. Check if 20% sprint capacity allocated to debt
3. Track debt reduction progress vs. targets
4. Identify critical debt requiring immediate attention
5. Report debt trends to Head of Engineering"
```

## Emergency Protocols

### Production Incident Response
```
"Gigi, activate incident response protocol:
1. Immediately assess production issue severity
2. Assemble emergency engineering response team
3. Implement immediate mitigation measures
4. Track incident resolution progress
5. Document post-mortem and prevention measures"
```

### Critical Bug Escalation
```
"Gigi, escalate critical engineering issue:
1. Immediately notify Head of Engineering
2. Assemble cross-functional response team
3. Prioritize fix over all other development
4. Coordinate emergency deployment if needed
5. Monitor fix effectiveness and user impact"
```
