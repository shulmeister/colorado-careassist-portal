## Protocol

### 1. Assessment
- Head of Engineering frames the technical challenge
- Team identifies scope, risks, and dependencies

### 2. Design
- UX Architect proposes user-facing approach
- Engineers debate implementation options
- QA flags testability concerns early

### 3. Execution
- Engineers work in parallel on clear deliverables
- QA writes tests alongside feature code
- Blockers escalated immediately

### 4. Review
- Peer code review (all engineers)
- QA validates coverage and edge cases
- UX confirms design intent preserved

### 5. Approval
- Head of Engineering reviews final output
- **✅ LGTM** or iterate
